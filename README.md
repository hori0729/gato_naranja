Herramienta simple de prueba de estrés para servidores HTTP que muestra resultados claros en consola.

## Características
- Ejecuta múltiples solicitudes concurrentes a una `URL` objetivo
- Calcula tasas de éxito/error, rendimiento y latencias (avg, p50, p95, p99)
- Sin dependencias externas (solo biblioteca estándar de Python)
- Al finalizar, muestra los resultados en la terminal

## Requisitos
- Python 3 instalado

## Uso
- Con URL directa:
```bash
python gato_naranja.py --target https://tu-servidor.com --requests 1000 --concurrency 50 --timeout 10
```
- Con IP/host y composición de URL:
```bash
python gato_naranja.py --target 192.168.1.10 --scheme http --port 80 --path / --requests 1000 --concurrency 50 --timeout 10
```
- Compatibilidad también con `--url` o `--ip`:
```bash
python gato_naranja.py --url https://tu-servidor.com --requests 1000 --concurrency 50 --timeout 10
python gato_naranja.py --ip 192.168.1.10 --scheme https --port 443 --path /api --requests 1000 --concurrency 50 --timeout 10
```

Al finalizar, se imprime un resumen con rendimiento, tasas de éxito/error, latencias y códigos de estado.

### Parámetros
- `--target` IP/host o URL completa. Si es URL (`http://`/`https://`) se usa tal cual; si es IP/host se compone con `--scheme`, `--port`, `--path`.
- `--url` URL objetivo (alternativa a `--target`).
- `--ip` IP/host objetivo (alternativa a `--target`).
- `--scheme` esquema `http`/`https` (por defecto `http`).
- `--port` puerto (por defecto `80`).
- `--path` ruta, por ejemplo `/api` (por defecto `/`).
- `--requests` número total de solicitudes (por defecto `100`).
- `--concurrency` número de hilos concurrentes (por defecto `10`).
- `--timeout` tiempo máximo por solicitud en segundos (por defecto `10`).

```text
  200: 50
```

## Notas
- Usa `https://` siempre que sea posible.
- Si tu servidor limita la tasa, ajusta `--requests` y `--concurrency`.#
